postal Address compiler






About



The process of converting an input file (stream of characters) into an understandable format is called syntax analysis. It is usually divided into two parts. The first, lexical analysis, tokenizes the input via a lexical scanner. The second, parsing, generates an abstract model of the tokens and then generates useful content from that model. I wrote a syntax analyzer using the flex tool for lexical analysis and the bison tool for parsing.

The program basically parses an input file that contains postal addresses and then outputs a representation of those addresses in XML.




Language/Technology Used



Linux, C, flex, bison




The Logic



There is a text input file that contains sets of address blocks. Each address block is separated from the next by exactly one blank line.

To scan/lexically analyze the input file, first, I wrote a .l (flex) file that has the token/grammar definitions using regular expressions. If characters in the file matched the regular expression, it returned a token indicating the existence of a certain word or character. Flex generates C code and I use the yylex() method to call the Flex-generated C code to get the tokens and output it to stdout.

To actually make sense out of the tokens/parse the file and convert it to XML format, I used Bison. I wrote a .y (bison) file following the BNF notation of a U.S Postal address. Bison looks at the tokens generated by flex and assigns it a meaning. For example, when it finds a token called SRTOKEN(Sr.) which is meant to be suffix, I embedded c code in there so that it outputs "<Suffix>Sr.</Suffix>" to an output file.
